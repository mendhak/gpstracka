Main Screen

Main Screen | Configuration | Advanced configuration

Verbose
Sometimes, the GPS logging may not work (inside a building, train, new country, alien invasion, etc). If you click verbose, the textbox will start displaying the raw GPS NMEA data being received by the receiver. If you understand it, then good for you, else it's something cool and complicated to show to your uninterested friends. But the main parameters you're looking for are GPGGA (location) and GPGSV (satellites visible). If you want to understand it, you can read up on NMEA sentence information. I also recommend you use this the very first time you run the application to figure out which COM port and Baud Rate is right for you.
Note: Verbose log is not available when using Windows Intermediate GPS driver.
Clear
This clears the textbox.
Settings
This takes you to the settings screen where you can specify the COM port, baud rate, and where to log the GPS data.
Exit
Quits the application. If you click this while GPS is active, you will be asked to quit. When logging to file, your data are never lost by terminating the aplication - correctly, or because of device shutdown or application crash.
Sattelites
Shows GPS sattelites information - signal strength, satellite visibility and location on sky. Sattelite infromation are actualized only when GPS device is actually running.
Start
Obviously, this starts the logging. If you selected Verbose in the options menu, it'll give you all data being received in the textbox. Else, it will log the data at the specified interval to the textbox or file.
If you have logged some data recently and you stopped logging and then you press start, you are asked to continue logging to the same file. This does not occure when you restart the application or visit the settings screen (a new file is created).
Stop
The Start button changes to Stop when you press Start. Then you can use it to stop GPS logging.

Configuration

Main Screen | Configuration | Advanced configuration

Comm Port
Setup source of GPS data
Use Windows driver
Check this to use Windows Intermediate GPS driver instead of eading data from COM port.
Using Windows driver may solve some problems with GPS.
Using COM port allows you to use GPS device not configured with Windows driver or use multiple GPS receivers at once.
You can also configure COM port in such way that it reads raw data from Windows driver.
Using Windows driver or reading data from Windows-driver-provided COM port allows you to run multiple GPS softwares at once, which is impossible when reading data directly from COM port GPS devices is attached to.
When using GPS driver NMEA data are not accessible. You cannot save NMEA log file and you cannot view verbose data.
Microsft GPS driver is configured in Start | Settings | System | External GPS.
Read
This drop down is the COM port that gives you your GPS data. This may be different on different phones. For me, it was COM1. You'll need to determine this for yourself by setting it and using Verbose mode.
If you are using another port, not listed in the drop down, then you can type it in anyways. Note: SOmetimes port may be named GPC instead of COM.
In case you have device with Microsft GPS Intermediate Driver (probably yes; its supported in Windows Embedded CE 6.0 and later and Windows Mobile 5.0 and later), the driver output port (as set up in registry) is displayed right of combo box as recomended. You should use it as it allows multiple GPS applications to wor in the same time. You can for example when riding a car, you can use a GPS navigation software (like iGO) for navigation and GPSTracka for logging.
Baud Rate
The data transfer speed. Different GPS receivers use different speeds, so you will need to experiment. I use 4800.
Polling Rate
How often the GPS data should be logged to textbox or file. I usually set this at 300 seconds.
Shorter polling interval causes more precise log. On the other hand shorter polling interval means bigger power consumption. Seting depends on speed you are going to move. For same precision you must set shorter interval when driving a car than while walking.
Logging Options - To Text Box
The best way to know that it's working is to keep this box checked, so that you can see the values printed to the textbox. When not checked you have no control if GPS is working or not.
Logging Options - To File
To log to file, tick this checkbox and then select a folder by clicking the "..." button. It's best if you create a "GPSLogs" folder in "My Documents" or on your Storage Card and then specify that path.
If not checked, you wil not have GPS log for future use! GPS logs are stored in files named by date and time when logging started.
Format GPX/KML/CSV
Some people prefer GPX (GPS XML format), some prefer KML (Google Earth XML format). If you want a lightweight file, then go for GPX; if you want to use your file with Google Earth, use KML (Google Earth can load GPX as well).
CSV format is simple text format where date, latitude, longitude and altitude are loged comma separated. It can be in detail configured in Advanced configuration.
Log altitude
Check this to log altitude informations as well. When not checked track / points are rendered on the ground.
Note: GPS altitude is somewhat different from altitude you can see in maps or Google Earth. This may cause the track to be rendered in mid-air or underground. You can easily instruct Google Earth to ignore track elevation (in case of points it's not as easy because you must change the setting for each point separatelly). The difference between GPS altitude and true altitude is caused by the way how it is measured by GPS and traditionally. GPS uses some unified ellipsoid (WGS84) model, while traditional measurement uses some local sea level whih can vary by the sea actually used. This difference is usually constant in one map and certain area and good map usually contains the informatin how its altitude differs from GPS altitude - you can then set altitude correction in advanced settings. Also keep in mind that GPS altitude is approximatelly 1.5 time accurate than longitude/latitude. In case your receiver supports it and you are using Windows GPS driver, you can instruct it in advanced settings to obtain true sea-level evelation instead of ellipsoid elevation.
Distinct points / track
The distinc points option logs just separate points to KML/GPX file, while Track logs connection line between them as well. Ignored for CSV.
Note: Points are always logged to KML, because KML does not allow description for track points. You can easily turn points on and off in Google Earth or even delete them as they are in single folder.
Save
Saves the settings to config file.
More
Shows the Advanced setting dialog when you can fine tune or completelly destroy ;-) the application.
Back
Resets settings to its previous state and goes to main screen.

Note: To reset settings completely navigate to application folder and delete the GPSTracka.exe.config file.

General

General | Display | KML | Formatting | Win GPS | Diagnostic | CSV

Time System / GPS
There are two possible sources of time informations, the GPS time provided by satellites and time provided by your device (set in control panel). GPS time is always acurate. GPS reports universal time (UTC), but the application can convert it for you to local time according to time zone set in Windows control panel.
Altitude correction
GPS altitude usually differes from altitude you can see in maps or Google Earth. This is caused by different style of measurement - GPS uses WGS84 ellipsoid. The difference is usualy constant for one map in certain area and good map gives you he information about the difference, so you can set-up correction here. Bad or no correction may cause your tracks and points to be rendered in mid-air or underground. You can turn altitude logging off as well (in settings screen). For yachting it is really not important.
Start immediatelly
By default GPSTRacka first waits for poling interval time before it records first point. Checking this forces the application to log first point immediatelly when you click Start.
Minimal distance
You may observe quite accurate GPS log while you are moving. But once you stop, you may see many points placed randomly around your actual location. You may prevent GPSTracka from logging points that are near from last logged location. This prevents multiple logs at/near same place, when you stop for a while. This sets up minimal distance between two points being logged. You should experiment a little with this. The right number depends on behavior of your GPS receiver, how fast you are moving and which precision of track log you want. 5 to 10 meters can be good option.
Pause after X invalid positions
Indicates number of invalid positions received from GPS in a row after which GPS device is closed and waits another polling interval period before trying to ead GPS data again.
Ignored after start until first valid position is got.
Because keeping GPS device running consumes some batery power, it's usefull to stop it when it does not provide any meaningful data (like when you are in a tunnel). On the other hand setting this to low number may cause that GPS device will never start reading data after leaving a tunnel, because the application will simply close it before it can find sattelites.
0 - never pause.
Keep awake devices
Comma-separated list of names of devices to keep awake when GPS is running.
Enlist device names such as GPS0 or GPS0:. Colon-terminated devices will be kept awake only during GPS cycle; other devices will be kept for whole time when GPSTracka is logging.
Unknown devices are skiped.
When not using Windows GPS diver, the COM port is kept awake automatically for whole time when GPSTracka is logging.

Formatting

General | Display | KML | Formatting | Win GPS | Diagnostic | CSV

Formatting is a little tricky process used by programmers in .NET to display date, time, numeric and other values in different formats. Specificly string formatting replaces each occurence of {x} with string representation corresponding object. x is decimal number - 0-based index of object. More, {x} can be in format {x,len}, {x:format} or {x,len:format}.

x
0-based index of object passed py application to formatting. Actual value depends on application.
len
Number - minimal lenght of string representation of the object. If actual string representation of the object is shorter than len it is paded with spaces to given length. If positive, the parameter is right-aligned; if negative, it is left-aligned.
format
This is the most important and most tricky parameter. It defines how the value is formatted. If omited default formatting (depends on object type and value and control panel settings) is used.
Actual valid value of format depends on type of object being passed to formatting.
Basically there are two types of formatting - predefined (single letter) and custom. For complete guide see .NET formatting guide.

For placemark and log formatting GPSTracka provides following values:

{0}
Date & time UTC
{1}
Longitude (positive for East, negative for West)
{2}
Latitude (positive for North, negative for South)
{3}
Altitude (if measured; 0 otherwise) in meters (see altitude notes above)
{4}
Date & time local (according to time zone set in Control panel).
{5}
Abbreviation of chosen altitude unit. Text only - do not apply any formatting after colon.

Note: When Time Zone in Control Panel is set incorrectly then:

  • When using time form GPS system, local time is computed incorrectly.
  • When using time from Windows, UTC time is computed incorrectly.

When you are planning trip across time zones, you'd better get time from GPS and log it as UTC time. - Unless you have as smart device as it switches time zons automatically as you cross the border :-).

General | Display | KML | Formatting | Win GPS | Diagnostic | CSV

Provides settings related exlusively to Microsft Windows Intermediate GPS Driver.

Use sea level altitude
Use seal level (the normal one) altitude instead of elipsoid one.
GPS system normally provides altidues measured to some kind of ellipsoid (WGS84 ellipsoid) instead of actual sea level as used in maps. Windows driver may provide sea-level altitude as well. If you are receiving no altitude after checking this, uncheck it.

Diagnostic

General | Display | KML | Formatting | Win GPS | Diagnostic | CSV

NMEA log
Check to create extra file in Save folder containing RAW GPS NMEA data. May be used for diagnostics. Contains almost the same data as written in verbose mode to text box. (In case the GPS device provide data too quickly the textbox skips some lines while file always contains all the data.)
Note: The file may grow easily.
Note: NMEA file is always named using system local time, because GPS time may not be available when it's being created.
Note: The file is not logged when using Windows Intermediate GPS driver.
Beep every X s
Makes device to beep every X seconds when GPS is running. 0 means do not beep.
Device dows not beep when suspended.
In addition beep count is show instatus bar which increments even when device is suspended.

CSV

General | Display | KML | Formatting | Win GPS | Diagnostic | CSV

This page controls settings related to CSV file format.

Separator
Character used as seperator of CSV fields. Default is comma (,); semicolon(;) or pipe (|) are often used.
TAB
Sets field separator to TAB character.
Text qualifier
CSV text qualifier used to suround values. Double (") or single (') quotes are commonly used.
From list chose when the qualifier should be used. Always, never or when needed.
When needed means that qualifier is used only when value of the field contains character used as separator.
In case that field contains character used as taxt qualifier and text qualifier is used, then is is escaped by writing it twice time.
Examples (seperator ,, qualifier " used as needed):
  • 142 becomes 142
  • 14"2 becomes 14"2
  • dc,x becomes "dc,x"
  • d"c,x becomes "d""c,x"
New line
Character(s) used as line separators. You can chose from common Windows, Unix and Apple seperators.
Header
Text used as header of CSV files. Empty and no header is written.
You are completelly responsible of corectness of header. It should reflect order of CSV fields and use the same rules for field separation and qualification as CSV data lines.
The TAB button insets the tab character at cursor position.
Available fields
Shows available CSV fields that are in use.
Use the >> button to add field among used ones.
Used fields
Shows fields used in CSV files generation. At least one field must be selected.
Use the << buton to remove field.
Use the ^ and v buttons to muve selected fiedl up (left) and down (right).
Date format
Format used for formating date into CSV file.
Date/time format described in Formatting is used, but without braces {} and numbers - only the part after colon (:).
Actual format can be tested using the ? button.
Unlike naming formats, this is evaluated in invariant culture instead of current culture.
UTC time
Checked when CSV file contaisn times in UTC coordinates, unchecked to use time in local time zone.
Note: You change the format so that it outputs the timezone.
